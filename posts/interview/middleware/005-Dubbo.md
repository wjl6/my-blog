## 1. dubbo是什么？和feign有什么区别？

Dubbo 是一个高性能、基于 Java 的 RPC 分布式服务框架，用来做“服务之间的远程调用”  
主要用于：
  - 服务注册与发现
  - 高性能远程调用
  - 服务治理（负载均衡、容错、限流等）  

核心目标只有一个：
  - 让分布式服务之间的调用，像本地方法调用一样简单、高效、可治理。

Feign 是一个声明式的 HTTP 客户端，用来简化微服务之间的 HTTP 调用。

| 维度    | Feign           | Dubbo          |
| ----- | --------------- | -------------- |
| 本质    | HTTP 客户端        | RPC 框架         |
| 协议    | HTTP            | TCP（Dubbo 协议等） |
| 性能    | 一般              | ⭐⭐⭐⭐ 高         |
| 序列化   | JSON            | 二进制            |
| 跨语言   | 好               | 一般             |
| 服务治理  | 依赖 Spring Cloud | 内置完善           |
| 使用复杂度 | 低               | 中              |
| 适用场景  | 对内 + 对外         | 内部服务           |

> Feign 更偏“HTTP 风格的服务调用”，适合对外接口或对跨语言友好的场景；  
> Dubbo 是高性能 RPC 框架，更适合企业内部高并发、低延迟的服务调用。
